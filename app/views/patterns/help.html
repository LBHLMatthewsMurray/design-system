Contextual help

Small overlays containing help content for complex interactions

Why have we created this pattern?

  • the GOV.UK pattern does not meet the user need

Relevant GOV.UK patterns

  • help text

Example component

Interaction principles

  • help should be available to users without interfering in their current flow
  • help should be easy to close, move and minimise until the user needs it
  • links to more comprehensive help should be easy to find, if contextual help doesn't answer a users question

Content principles

  • refer to GOV.UK pattern

Accessibility principles

  • needs testing

User research

  • users find side-by-side guidance more helpful and easier to use.
  • small overlay windows offer a smallest interference with the users flow
