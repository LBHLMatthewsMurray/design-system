Using the patterns in your project

We encourage everyone to use the patterns as part of their project, to create consistency in services but also to gain better user feeback on them so we can iterate and improve.

Sketch file

Use the sketch file as are starting point for your designs, but consider feeding back any developments or changes you make, to the community.

Include in your html

You can link directly to the css file to include or you can include as a npm sub-module.

Linking directly to the heroku application isn't recommended as it may change or be down without warning, but you can include using

https://home-office-digital-patterns.herokuapp.com/public/stylesheets/application.css

Include in the prototyping kit

Update your package.json file.

  • this repository will install as a node dependancy in their package.json file. The styles and assets will then be available in node_modules/hod-patterns

import the sass files required into your stylesheet

  • You may need to setup a symlink from the folder
  • setup a build process using the same processors as this repository

Example of how to include the package in package.json

{
  "devDependencies": {
    "hod-patterns": "https://github.com/hod/hod-patterns.git#v1.0.0",
  }
}
